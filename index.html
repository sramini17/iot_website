What is a Server Farm?

A server farm is a group of computers, usually maintained by an organization to supply server functionality far beyond the capability of a single machine.

Server farms often consist of computers which require a large amount of power to run and to keep cool. At the optimum performance level, a server farm has enormous costs (both financial and environmental) associated with it. Therefore, it is very important to have a tool to monitor the physical conditions in the location used to store the computers, else one runs the risk of damaging expensive equipment and having suboptimal server performance.

The Server Farm Monitor

Server Farm Monitor (SFM) is a device that monitors the temperature and humidity in such location and it uploads the data to a web server that can be accessed from anywhere in the world. It will also send email notifications when the conditions being monitored exceeds a certain threshold. Implementing such device in the Cal Poly Computer Science Lab (CSL) can prevent server downtime and improve the quality and lifespan of the servers.

Technical Details

The image above describes the program flow and is described in more detail below.
F-1. Data Acquisition via DHT11 Sensor
Uses an open source C driver from DHT to read GPIO Pin. The Python script reads the temperature and humidity every 2 seconds and uploads the reading to a MySQL database on a remote server. The Python script also runs upon start up of the Raspberry Pi.

F-2 - 4: Presenting the data
Click HERE to go to the dashboard.
Got a Domain name from GoDaddy for the dashboard's website which displays the current temperature and humidity, the averages of the temperatures and humidity over time, and the current date and time. The dashboard website uses HTTP Server with Dashing on Ruby Sinatra. All the notification that exceed or are below a threshold are sent from sfmnotifications0@gmail.com to user's email.

Conclusion

Technical Challenge

There were imperfect readings from the sensor (e.g. ludicrous spikes in humidity and temperature). This issue was solved by using the weighted running average to calculate humidity and temperature over time, so that the notifications weren't being sent for the the imperfect readings.

Future Work

In the future, using multiple sensors instead of just one sensor would be a good idea. This would be to get a better idea of temperature and humidity throughout the server room. Also putting sensors in different locations in the server room would be a good idea. For example putting sensors next to air conditioning units to detect whether they are functional.

In conlcusion...

The SFM is succesful in detecting alarming changes in temperature and humidity and contacts server admins so that they may act accordingly.
